# Cue Generation Design

This document defines the target cue behavior for Cuemaster Playbooks generated by Stager. The manifest shape remains defined in `planning/specs/playbook_manifest.md`.

## Scope

Cue generation answers one question: for each rehearsable role response, what text and audio should Cuemaster play before the actor speaks?

For schema version 1, every rehearsable non-meta role line must have:

- One required cue payload.
- One required response payload.
- Required cue audio and response audio.

`_NARRATOR`, `_CALLER`, and `_ANNOUNCER` are not rehearsable roles by default.

## Current Stager Cue Media Behavior

`CueBuilder` currently builds role-specific MP4 cue tracks with chapter markers.

Current behavior:

- The response is the selected role's segment audio for a role block.
- The default prompt is the preceding non-meta role block in the same part.
- Prompt audio can be prefixed with the previous speaker's callout audio.
- Prompt audio can be cropped to keep cue media short.
- A configurable delay is inserted before and after responses.
- Chapters use `CUE <role> <segment_id>` and `LINE <role> <segment_id>` titles.

The MP4 cue track is useful as a Stager output for actors using general-purpose audio players, but Cuemaster Playbooks should not depend on MP4 chapter parsing or `CueBuilder` internals. Playbooks should encode cues and responses directly in `manifest.json`.

## Target Playbook Cue Rules

For each rehearsable role block:

- Use the immediately preceding cue-eligible block in the same part as the cue when one exists.
- Prefer preceding spoken role audio when it exists.
- If the preceding cue-eligible block is a top-level direction, description, title, or inline direction-only narrator segment, use narrator text and narrator audio.
- If no preceding cue-eligible block exists in the same part, fall back to the part title, then the play title.
- Do not use callout audio as cue audio.
- Do not emit a rehearsable line with `cue: null`.
- Do not emit a rehearsable line with missing cue audio.
- Do not emit a rehearsable line with missing response audio.

Cue-eligible block order follows parsed play order, not markdown output order.

## First-Line Behavior

The first rehearsable line in a part still needs a cue.

Rules:

- If a preceding narrator/title/direction block exists in the same part, use it as the cue.
- If no preceding cue-eligible block exists in the same part, use the current part title as the cue.
- If the current part has no title, use the play title as the cue.
- Do not borrow a cue from a previous part.
- Do not omit a rehearsable line just because it is the first line in a part.
- Do not synthesize silence or placeholder cue audio.

This strict behavior keeps Cuemaster simple: every actor line is present, and every line can assume a cue exists.

## Directions

Stage directions are context by default, but they become playable cues when they are the preceding cue-eligible block.

Rules:

- Top-level `DirectionBlock` and `DescriptionBlock` cues use `_NARRATOR` segment audio.
- Inline directions stay separate from spoken response text.
- If an inline direction is the only preceding cue content, its narrator audio is required.
- Direction text should be preserved in the manifest as structured direction/context text.

## Caller And Announcer Audio

Caller and announcer snippets are not cue audio.

Rules:

- Caller/callout audio may be included as optional playback assets when Stager exports it.
- Cuemaster may expose a preference to play callouts before cues.
- Missing callout audio should only fail when callouts are explicitly enabled as required.
- Announcer and Librivox preamble/epilog snippets are outside actor rehearsal cue generation.

## Legacy Stager MP4 Cue Markers

Stager can generate role-specific MP4 cue files. Those MP4 files use chapter markers as seek points inside a single audio file:

- `CUE <role> <segment_id>` starts at the prompt audio, not the callout.
- `LINE <role> <segment_id>` starts at the response audio.
- Delays and callouts are not separate MP4 chapter markers in the initial behavior.
- When prompts are disabled, only `LINE` MP4 chapter markers are emitted.

Cuemaster Playbooks should not use MP4 chapter markers. They should use manifest cue and response records.

Do not treat legacy MP4 cue generation as the implementation substrate for Playbooks. Both outputs should reuse shared play parsing and segment audio files, but each output should have its own assembly layer.

## Deterministic Test Fixtures

Cue-generation tests should:

- Build tiny WAV fixtures in `tmp_path`.
- Use `PathConfig` pointing at `tmp_path`.
- Avoid real recordings and downloaded models.
- Assert audio duration is greater than zero for roles with fixture audio.
- Assert exact MP4 chapter marker titles and start/end ordering for legacy Stager cue media.
- Assert missing required cue or response audio raises an exception once strict Playbook generation exists.
